AudioContext Interface

  • Creates all AudioNodes
  • Contains the routing graph
  • AudioDestinationNode is the final node
  • Usually only a single AudiContext is used

AudioNode Interface

  • Building block of an AudioContext
  • Parent class for all input/output and processing nodes
  • Multiple nodes can be connected as inputs
  • Output can be connected to multiple nodes (fan-out)

Scheduling Audio Events

  • Audio events are relative to the context's currentTime
  • currentTime starts at zero on context creation and increases in real-time
  • currentTime cannot be stopped or paused

OscillatorNode Interface

  • Generates a periodic waveform
  • Types: sine, square, sawtooth, triangle, custom

Example: Oscillator

  • Create an AudioContext
  • Create OscillatorNode and set waveform type
  • Connect the OscillatorNode to the context's destination

Audio Context Clock

  • is a float
  • starts at zero on context creation
  • increases in real-time
  • cannot be stopped or paused
  • all audio events are relative to this clock
